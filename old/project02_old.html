PHOTO GALLERY

Slim Framework, jQuery & AJAX

WHAT IT DOES

The gallery app created has a login and registration functionality that is fully validated and allows users to view, edit, add and delete photos on a personal photo gallery. Once a user is registered and logged in they can fill up their gallery with images. When the image is added with a title and caption it is added to their gallery list. The added image saves the users Id so that the gallery is unique to them, they won’t be able to view other people’s images and other people can’t view theirs. Should the list of photos get too long and the user wishes to find a specific photo they can use the search bar to find the image by name, and the reset button will make all the images reappear.

HOW IT WORKS

The data for the application is stored on a MySQL server and is split into two tables. A table for the users and a table for the photos. The photos table uses a foreign key to the user table, so that when a user logs in they only see photos they’ve submitted. Slim is implemented as a dispatcher that manages HTTP requests and responses. Slim Framework is basically a PHP micro framework that allows you to write simple but powerful web applications and API’s. AJAX is used so that the gallery can update its content without reloading web page. Through Slim, it requests and receives data asynchronously without disrupting the display and behaviour of the web page.

WHAT I LEARNED

Through the creation of the photo gallery, I learned how to implement a MySQL database that utilises the Slim Framework. Using AJAX and jQuery in conjunction with Slim, the web application dynamically updated whenever data off the server was changed. The gallery was given basic view, add, update, delete and search functionality and login and registration functionality was created with full validation. Finally, using keys being stored in the session storage gave the web application the ability to allow users to only view images relating to their user id.
